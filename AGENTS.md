# Project Notes for Codex

This app is already used in production. Keep changes small, reversible, and verified.

## Stack

- Vite 5, React 18, TypeScript 5
- Tailwind CSS v3 and shadcn/ui
- Supabase: Postgres, Auth, Storage, Edge Functions in Deno
- React Router v6, React Query, Context API

## Commands

- Install: `npm install`
- Dev server: `npm run dev` (port 8080)
- Production build: `npm run build`
- Lint: `npm run lint`
- Type check: `npx tsc --noEmit --pretty false`

## Critical Rules

- Timezone handling must use Europe/Madrid. Prefer the existing `formatMadridDate` patterns.
- Durations are handled in hours with 0.25 increments; internally, use minutes when the current code does.
- For new public Supabase tables, add explicit grants, enable RLS, and create matching policies.
- Use `ON DELETE SET NULL` for foreign keys that point to `tasks` unless the product requirement says otherwise.
- After changing Edge Function logic, redeploy the affected function.
- Do not manually edit `src/integrations/supabase/types.ts`; it is autogenerated.
- Mobile file inputs need the existing absolute-position input pattern, forced `image/jpeg`, and explicit upload `contentType`.
- Never put Supabase `service_role` keys in frontend code, committed files, or Vite `VITE_*` variables.

## Production Safety

- Work on feature branches, not directly on `main`.
- Before schema changes, inspect existing migrations and RLS policies.
- Keep migrations explicit and reviewable.
- Verify with at least `npm run build` and `npx tsc --noEmit --pretty false` before shipping.
